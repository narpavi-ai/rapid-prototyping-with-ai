# Resources

## Tools used in this workshop

- [Lovable](https://lovable.dev) — the no-code AI app builder used from
  Stage 3 onward
- [ChatGPT](https://chatgpt.com) or [Claude](https://claude.ai) — used in
  Stages 1-3 to think through the problem, blueprint, and first prompts

## Official Lovable documentation referenced in this repo

All verified live against the current docs on **July 5, 2026** — Lovable
ships fast, so re-check these if you're reading this much later.

- [docs.lovable.dev](https://docs.lovable.dev) — main documentation site
- [Knowledge files](https://docs.lovable.dev/features/knowledge) — the
  persistent project-context concept used in
  [Stage 3](stages/03-lovable-prompt)
- [Build mode (formerly Agent mode)](https://docs.lovable.dev/features/agent-mode) —
  Lovable's autonomous execution mode, used in
  [Stage 4](stages/04-build-in-lovable)
- [Plan mode](https://lovable.dev/faq/ai-agent/plan-mode) — Lovable's
  reasoning-only mode, used throughout
  [Stage 4](stages/04-build-in-lovable) and
  [Stage 5](stages/05-test-iterate)
- [Best practices](https://docs.lovable.dev/tips-tricks/best-practice) —
  the incremental build workflow behind
  [Stage 4](stages/04-build-in-lovable)
- [Learn to Prompt (Lovable Academy)](https://academy.lovable.app/academy/prompting) —
  the five prompting principles referenced in
  [Stage 3](stages/03-lovable-prompt)
- [GitHub integration](https://docs.lovable.dev/integrations/github) — the
  two-way sync setup flow in [Stage 6](stages/06-scale-up)
